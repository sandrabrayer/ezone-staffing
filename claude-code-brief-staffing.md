# Claude Code Brief — E-ZONE אפליקציית ניהול כוח אדם · חיבור לענן ופריסה

> העתק את כל הקובץ הזה ל-Claude Code כנקודת פתיחה. זהה במסלול ל-outpatient.

---

## רקע

קיים פרוטוטיפ עובד של אפליקציית ניהול כוח אדם (`ezone-staffing.html`). כרגע הנתונים נשמרים ב-localStorage בלבד. המשימה: להפוך אותו לאפליקציית ענן מלאה, באותו stack ובאותו מסלול כמו האפליקציות הקודמות של E-ZONE (outpatient וכו').

**משתמשת בלעדית:** מורן, משאבי אנוש.

**4 בתים קבועים:**
- `ramot` — רמות השבים — מנהל: שחר
- `asher` — רעננה אשר — מנהל: עידו
- `ofroni` — קיסריה עפרוני — מנהל: חנן
- `rehab` — קיסריה ריהאב — מנהל: רנטה

---

## מה האפליקציה עושה (לוגיקה קיימת לשמר)

### לכל בית בנפרד
מצבת עובדים עם: שם, תפקיד, שכר ברוטו חודשי, חלקיות משרה (%), הערות.
- עלות בפועל = `round(salary * pct/100)` — מחושב בצד הלקוח, לא נשמר ב-Sheet.
- כרטיסי סיכום לבית: מספר עובדים, עלות שכר חודשית משוקללת, סה״כ ברוטו במשרה מלאה, חלקיות ממוצעת.
- דגל "כיסוי" ויזואלי כשההערה מכילה: כיסוי / מחליף / מחליפה.

### מבט כללי מרכז
- סה״כ עובדים, עלות שכר כוללת, פירוט כרטיס לכל בית, מספר העברות.
- היסטוריית העברות מלאה (לא נמחקת לעולם).

### העברת עובד בין בתים
מסיר מהבית המקורי, מוסיף לבית היעד, ורושם רשומת היסטוריה עם:
שם, מאיזה בית, לאיזה בית, סוג סיבה (כיסוי חוסר / העברה קבועה / צורך תפעולי / אחר), פירוט חופשי, תאריך.

## משימות לביצוע

### 1. מבנה Google Sheet
החלטה הושארה לשיקולך — ההמלצה: **טאב לכל בית + טאב `history`**.

טאב בית (למשל `ramot`), כותרות שורה 1:
```
id | name | role | salary | pct | notes
```

טאב `history`, כותרות שורה 1:
```
timestamp | name | from_house | to_house | reason_type | reason | date
```

**להתחיל ריק** — רק שורות כותרת, בלי נתוני דוגמה. מורן תזין את כל המצבת מאפס.

### 2. Apps Script (Web App)
- `doGet` — מחזיר JSON עם כל הבתים + history.
- `doPost` — פעולות: `addEmployee`, `updateEmployee`, `deleteEmployee`, `moveEmployee`.
- `moveEmployee` מבצע אטומית: מחיקה מטאב המקור, הוספה לטאב היעד, append לטאב history.
- מפתח סוד (shared secret) שנשלח בכל בקשה ומאומת ב-Apps Script — לא חושפים את ה-URL לבד.

### 3. server.js — פרוקסי Express
- מגיש את `public/` (ה-HTML).
- `/api/data` ו-`/api/action` — פרוקסי ל-Apps Script. ה-secret וה-URL ב-environment variables בלבד, אף פעם לא בקוד צד-לקוח.
- בלי לחשוף את כתובת ה-Apps Script לדפדפן.

### 4. PIN gate
- **PIN אחד למורן בלבד.** מסך נעילה לפני האפליקציה.
- ה-PIN מאומת בצד שרת (לא hardcoded ב-HTML). שמירה ב-env var.
- session פשוט (cookie/localStorage token) כדי לא להקליד בכל רענון.

### 5. החלפת שכבת הנתונים ב-HTML
- להחליף את `load()`/`save()` של localStorage בקריאות `fetch` ל-`/api/data` ו-`/api/action`.
- לשמר את כל ה-UI וה-RTL כפי שהם.
- מצב טעינה + טיפול בשגיאות רשת (הודעה ברורה למורן בעברית).

### 6. פריסה — Railway
- repo נפרד (לא בתוך repo קיים).
- environment variables: `APPS_SCRIPT_URL`, `SHARED_SECRET`, `MORAN_PIN`.
- בדיקת בריאות בסיסית.

## הנחיות קבועות (חובה)

1. **תיעוד לכל שינוי ולכל commit** — הודעות commit ברורות + עדכון `CHANGELOG.md`.
2. **שיטות אבטחה מיטביות** — secrets ב-env בלבד; אימות secret ב-Apps Script; PIN בצד שרת; אין מפתחות בקוד צד-לקוח; ולידציה של קלט בצד שרת.
3. **בדיקות** — בדיקות לפונקציות הליבה: חישוב עלות משוקללת, לוגיקת `moveEmployee` (כולל שלמות ההיסטוריה), ולידציה.
